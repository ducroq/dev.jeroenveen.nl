# Where do future seniors come from?

*Status: seed note, not yet drafted. Captured 2026-05-12 from a chat reaction to a breathless LinkedIn post about "Agentic Consulting Mega Companies" (OpenAI's "Deployment Company" and Anthropic + Blackstone/Goldman/H&F consortium). The post argued the consulting pyramid collapses to "1 partner + agent orchestrator + agent swarm, everyone else gone." It dropped the part where the analyst pyramid is also a **training pipeline**, which is where the seed question came from.*

---

## The seed

The senior-trust article ("Senior developers trust AI less than juniors", post 2 of the WHY arc) names *who* validates today: seniors, because they have the pattern library to recognise plausible-but-wrong output. It does not ask the next question:

**Where do future seniors come from when the slow work that built today's library is now done by agents?**

Every "AI replaces juniors / analysts / [tier-N role]" argument quietly assumes seniors are already there. The pattern library is treated as a stock, not a flow. But it is a flow, and the practices that produced it (slow debugging, reading raw output before any summary, sitting overnight with a bug, reproducing a calculation by hand) are exactly the practices augmented practice tends to absorb first because they are the expensive ones.

This is not "AI is bad for juniors." It is structural: the activity that built senior judgement was the slow, painful activity. If agents do that activity, the path to senior judgement narrows.

## Three possible homes

Ordered by preference at the moment of capture:

1. **AE pattern essay (new pattern).** Sibling to *Reproduce, Don't Assess*. That pattern is about validating the agent's output; this one is about building the human who can later validate. Working name candidates:
   - *The Apprenticeship Asymmetry*
   - *Library-Building vs. Library-Consuming*
   - *Where do future seniors come from?* (question as title, pattern named more dryly inside)

   The pattern itself: augmented practice has to consciously preserve the activities that *build* pattern recognition (slow debugging, reading raw output, reproducing by hand), because the default is for AI to absorb exactly those activities. Without conscious preservation, the apprenticeship that produces senior judgement is foreshortened.

2. **Paragraph inside WHY post 3 (HAZOP / domain bias).** The future-seniors thread is a natural cousin: both ask "who can see what the agent can't?" Bolting it in could thicken post 3.

   Risk: post 3 is already carrying validation-independence. Adding career-pipeline-decay risks making it two essays in one.

3. **Standalone short LinkedIn post.** Something like:

   > Every "AI replaces juniors" argument assumes seniors are already there. Where do the next seniors come from?

   Tight, provocative, low investment, would draw comments. Could function as a teaser pointing at (1) once the AE pattern exists.

## Open questions before drafting

- Has *Reproduce, Don't Assess* or another AE pattern already absorbed apprenticeship-language? Check the AE pattern library before naming a new pattern.
- Are there analogue domains where this argument has been made already, and lost or won? Chess engines and grandmaster training is one (mixed evidence, some argue modern GMs are positionally stronger and calculatively shallower). Medical residents using point-of-care tools vs. memorising Harrison's is another. Aviation crew-resource-management after autopilot is a third.
- The METR 19%-slower-for-seniors and JetBrains senior-trust data are about *today's* seniors. The future-seniors question needs a different evidence base (longitudinal cohorts, apprenticeship outcomes, or at minimum analogue precedents from other domains). Without that, the piece risks being a worried op-ed dressed as an essay.

## Reusable phrasings

- *"Where do future seniors come from?"*: strongest line. Question form, plain English, lands immediately, works as title / hook / closing all at once. Jeroen reacted to this one first, which is the relevant signal.
- *"every 'AI replaces juniors' argument assumes seniors are already there"*: opening filter for the short LinkedIn version.
- *"the activity that built senior judgement was the slow, painful activity"*: travels to any context where someone proposes compressing the painful part.

**Register check, dropped from the candidate list:** *"the pattern library is a flow, not a stock"* imports econ vocabulary (stock vs. flow) that was not in the original frame. Sounds sharp on first read but is exactly the kind of tidy management-book label the writing-guide flags. Keeping it documented here so it does not creep back in by the time drafting starts.

## Decision

Hold as a seed until either (a) the AE pattern library is reviewed for existing apprenticeship-language, or (b) post 3 of the WHY arc is drafted and we can decide whether to absorb the angle or keep it standalone.
